## 2026-05-04 - Avoid chained array methods on hot paths
**Learning:** Chaining `.map().filter()` or similar array methods on frequently executed paths (like account selection in `HybridStrategy` and `BaseStrategy`) creates intermediate arrays and wrapper objects (e.g., `{ account, index }`) that are immediately discarded. This causes excessive object allocations and increases garbage collection (GC) pressure, which can degrade overall application performance.
**Action:** Replace chained array methods with single-pass `for` loops. Evaluate conditions simultaneously and push directly to a result array, or use pre-allocated arrays (`new Array(length)`) when the final size is known. This bypasses intermediate allocations and minimizes GC overhead without changing the core business logic.
