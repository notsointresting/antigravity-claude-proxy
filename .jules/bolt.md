## 2024-03-14 - [Optimize Account Candidate Selection]
**Learning:** Found a performance bottleneck specific to the `HybridStrategy.#getCandidates` logic where multiple array allocations and `.map().filter()` chains were used to evaluate accounts against various criteria (basic usability, tokens, health, quota). This creates unnecessary intermediate objects and GC pressure, especially since the tests check each criterion in a "waterfall" fallback manner.
**Action:** Replaced the multi-pass `.map().filter()` approach with a single-pass `for` loop that evaluates all criteria sequentially and populates the `candidates`, `fallback`, `emergency`, and `lastResort` arrays directly. This reduces GC overhead and significantly improves performance without altering the core logic. Next time, prefer single-pass loops over multiple array iterations for complex filtering logic.
