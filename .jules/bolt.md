## 2026-05-29 - Array methods performance on hot paths
**Learning:** In highly trafficked pathways (like array manipulations per request message/block), chaining JavaScript's functional array methods (`map().filter()`, `filter().map()`, `sort()`) incurs significant performance penalties due to intermediate array creation, garbage collection, and loop overhead. Additionally, finding the max score with `.map(calc).sort()[0]` has O(N log N) time complexity and O(N) space complexity.
**Action:** Replace functional chains with single-pass `for` loops in hot paths, and use a simple linear scan (`for` loop with `maxScore` tracking) to find the best candidate in O(N) time and O(1) space. Use fast-path assertions (`Array.some()`) to return original array references unmutated when modifications are not needed.
