## 2026-03-21 - Optimize HybridStrategy account selection loop\n**Learning:** The `#getCandidates` method in `HybridStrategy` iterated over the same `accounts` array multiple times using chained `.map().filter()` operations, creating several intermediate objects and arrays during every request. By evaluating conditions across the same dataset, we can compute disjoint categories (like candidates, fallback, emergency, lastResort) efficiently in a single pass.\n**Action:** Replaced chained `.map().filter()` operations with a single `for` loop to significantly reduce GC overhead and avoid temporary object allocations on hot paths like request routing.
