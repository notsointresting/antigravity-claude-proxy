## 2026-05-22 - Optimize array operations with single-pass categorization
**Learning:** In code paths evaluated frequently (hot paths) like `HybridStrategy` account fallback categorization, chaining `.map().filter()` causes unnecessary garbage collection from intermediate array allocations. Replacing these with a single-pass `for` loop that cascades entries into priority collections simultaneously avoids the intermediate array allocations while preserving cascading fallback sequences (e.g., normal -> quota -> emergency -> lastResort). Additionally, sorting an array (`O(n log n)`) just to get the max value is inefficient and can be replaced with a single `O(n)` tracking scan.
**Action:** When filtering objects into fallback buckets on hot paths, perform evaluations within a single `for` loop to categorize elements rather than running multiple `.map().filter()` passes. When searching for the single best candidate, manually track the max item rather than sorting the collection.
