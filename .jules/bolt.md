## 2026-05-06 - Replacing Map/Filter/Sort Chains with Pre-allocated Loops
**Learning:** In hot execution paths within JavaScript classes handling large arrays or frequently running routines (like `BaseStrategy.getUsableAccounts` and `HybridStrategy.#getCandidates`), chaining functional array methods like `.map().filter()` causes severe intermediate object allocations (specifically Wrapper objects like `{ account, index }`) and puts strain on the Garbage Collector. Sorting a newly constructed array using `.sort()` is typically $O(n \log n)$ but often unnecessary when only the best candidate is needed.
**Action:** Replace these chains with pre-allocated arrays (`[]` filled by index or pushed to) and a single-pass `for` loop. If only the maximum/best element is required (as in the scoring loop), keep track of the `best` element directly within the loop instead of accumulating objects into an array and running an expensive sort operation.
